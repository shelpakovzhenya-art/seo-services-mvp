# 🚀 Быстрое исправление после деплоя

## ✅ Что нужно проверить и исправить:

### 1. Проверьте, что деплой прошел успешно

1. Откройте [Vercel Dashboard](https://vercel.com/dashboard)
2. Найдите ваш проект
3. Проверьте последний деплой - должен быть статус "Ready" (зеленый)
4. Если деплой не прошел - нажмите "Redeploy"

### 2. Настройте переменные окружения в Vercel

**Обязательно проверьте эти переменные:**

1. В Vercel Dashboard → Settings → Environment Variables
2. Убедитесь, что есть:
   ```
   DATABASE_URL=postgresql://user:password@host:port/database
   SITE_URL=https://seo-update.ru (или ваш домен)
   ADMIN_USER=admin
   ADMIN_PASS=admin123
   ```

3. Если переменных нет - добавьте их
4. После добавления - **перезапустите деплой** (Redeploy)

### 3. Заполните базу данных (если меню не появилось)

**Вариант A: Через Vercel CLI (рекомендуется)**

```bash
# Установите Vercel CLI (если еще не установлен)
npm i -g vercel

# Войдите в Vercel
vercel login

# Подключитесь к проекту
vercel link

# Получите переменные окружения
vercel env pull .env.local

# Установите DATABASE_URL
export DATABASE_URL="your_postgresql_url_from_vercel"

# Запустите seed
npx tsx prisma/seed.ts
```

**Вариант B: Через Vercel Dashboard (проще)**

1. Откройте Vercel Dashboard → ваш проект
2. Перейдите в **Settings** → **Functions**
3. Найдите функцию или создайте новую
4. Или используйте **Vercel CLI** как выше

**Вариант C: Через админ-панель (если можете войти)**

1. Войдите в админку: `https://your-site.vercel.app/admin/login`
2. Перейдите в **Настройки** → **Меню**
3. Создайте пункты меню вручную:
   - Главная: `/`
   - Услуги: `/services`
   - Контакты: `/contacts`
   - Кейсы: `/cases`
   - Отзывы: `/reviews`
   - Блог: `/blog`
   - Калькулятор: `/calculator`

### 4. Настройте домен (если ссылка не изменилась)

1. В Vercel Dashboard → Settings → Domains
2. Добавьте домен: `seo-update.ru`
3. Настройте DNS записи у регистратора домена
4. Дождитесь распространения DNS (1-24 часа)
5. После этого обновите `SITE_URL` в переменных окружениях

### 5. Принудительный редеплой

Если изменения не применились:

1. В Vercel Dashboard → Deployments
2. Найдите последний деплой
3. Нажмите "..." → "Redeploy"
4. Или сделайте пустой коммит:
   ```bash
   git commit --allow-empty -m "Force redeploy"
   git push
   ```

## 🔍 Проверка после исправления:

### Проверьте меню:
1. Откройте сайт
2. Проверьте шапку - должно быть меню с разделами
3. Проверьте футер - там тоже должно быть меню

### Проверьте домен:
1. Если настроили `seo-update.ru` - проверьте, что он работает
2. Проверьте SSL сертификат (должен быть зеленый замочек)

### Проверьте базу данных:
1. Войдите в админку
2. Проверьте, что данные есть
3. Проверьте меню в настройках

## ⚡ Быстрая команда для всего:

```bash
# 1. Получить переменные окружения
vercel env pull .env.local

# 2. Установить DATABASE_URL
export DATABASE_URL="your_postgresql_url"

# 3. Заполнить базу данных
npx tsx prisma/seed.ts

# 4. Проверить, что все работает
npm run build
```

## 📝 Чек-лист:

- [ ] Деплой прошел успешно (статус "Ready")
- [ ] Переменные окружения настроены (DATABASE_URL, SITE_URL, ADMIN_USER, ADMIN_PASS)
- [ ] База данных заполнена (seed выполнен)
- [ ] Меню появилось на сайте
- [ ] Домен настроен (если нужно)
- [ ] SSL сертификат работает

## 🆘 Если ничего не помогло:

1. **Проверьте логи Vercel:**
   - Dashboard → Deployments → последний деплой → Logs
   - Ищите ошибки

2. **Проверьте подключение к базе данных:**
   - Убедитесь, что DATABASE_URL правильный
   - Проверьте, что база данных доступна

3. **Проверьте консоль браузера:**
   - F12 → Console
   - Ищите ошибки

4. **Свяжитесь со мной** с логами ошибок

---

**Главное:** После изменения переменных окружения в Vercel - обязательно сделайте **Redeploy**!
