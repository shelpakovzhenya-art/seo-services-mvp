# 🔐 Исправление проблемы с входом в админку

## ✅ Что исправлено:

1. **Улучшена обработка ошибок** в `lib/auth.ts` - добавлено детальное логирование
2. **Улучшена валидация** в `app/api/auth/login/route.ts` - обрезка пробелов, лучшие сообщения об ошибках
3. **Создан скрипт сброса пароля** - `scripts/reset-admin.ts` для восстановления доступа
4. **Улучшен seed** - правильная обработка username и password

## 🔧 Как исправить проблему:

### Вариант 1: Сбросить пароль админа (рекомендуется)

**Локально:**
```bash
npx tsx scripts/reset-admin.ts admin admin123
```

**В продакшене (через Vercel CLI):**
```bash
# Получите переменные окружения
vercel env pull .env.local

# Установите DATABASE_URL
export DATABASE_URL="your_postgresql_url"

# Сбросьте пароль
npx tsx scripts/reset-admin.ts admin admin123
```

**Или с кастомными данными:**
```bash
npx tsx scripts/reset-admin.ts ваш_логин ваш_пароль
```

### Вариант 2: Проверить, что админ существует в базе

**Локально через Prisma Studio:**
```bash
npx prisma studio
```

Откройте таблицу `Admin` и проверьте:
- Есть ли запись с username `admin`
- Правильно ли захеширован пароль (должен начинаться с `$2a$` или `$2b$`)

### Вариант 3: Пересоздать админа через seed

**Локально:**
```bash
# Убедитесь, что .env настроен
# DATABASE_URL="file:./prisma/dev.db"
# ADMIN_USER=admin
# ADMIN_PASS=admin123

npx tsx prisma/seed.ts
```

**В продакшене:**
```bash
# Установите DATABASE_URL
export DATABASE_URL="your_postgresql_url"
export ADMIN_USER=admin
export ADMIN_PASS=admin123

npx tsx prisma/seed.ts
```

## 🔍 Проверка работы:

1. **Откройте страницу входа:** `/admin/login`
2. **Введите:**
   - Логин: `admin`
   - Пароль: `admin123`
3. **Проверьте консоль браузера** (F12) на наличие ошибок
4. **Проверьте логи сервера** на наличие ошибок

## 🐛 Отладка:

### Если все еще не работает:

1. **Проверьте переменные окружения:**
   ```bash
   echo $ADMIN_USER
   echo $ADMIN_PASS
   ```

2. **Проверьте подключение к базе данных:**
   ```bash
   npx prisma db pull
   ```

3. **Проверьте логи:**
   - В консоли браузера (F12 → Console)
   - В логах Vercel (Dashboard → Deployments → Logs)

4. **Проверьте, что админ существует:**
   ```bash
   npx prisma studio
   # Откройте таблицу Admin
   ```

## 📝 Стандартные учетные данные:

- **Логин:** `admin`
- **Пароль:** `admin123`

**⚠️ ВАЖНО:** После первого входа смените пароль на более безопасный!

## 🔄 Автоматическое создание админа:

При каждом деплое на Vercel автоматически выполняется `prisma/seed.ts`, который создает/обновляет админа с данными из переменных окружения:
- `ADMIN_USER` (по умолчанию: `admin`)
- `ADMIN_PASS` (по умолчанию: `admin123`)

Убедитесь, что эти переменные настроены в Vercel Dashboard → Settings → Environment Variables.
