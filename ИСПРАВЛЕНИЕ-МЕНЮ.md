# 🔧 Исправление проблемы с меню на сайте

## ✅ Что исправлено:

1. **Добавлено fallback меню** - если база данных пустая, используется дефолтное меню
2. **Улучшен seed скрипт** - гарантированно создает все пункты меню
3. **Улучшен setup.js** - seed выполняется в продакшене даже при ошибках
4. **Добавлено логирование** - видно, когда используется дефолтное меню

## 📋 Структура меню:

Меню должно содержать следующие разделы:
- **Главная** (`/`)
- **Услуги** (`/services`)
- **Контакты** (`/contacts`)
- **Кейсы** (`/cases`)
- **Отзывы** (`/reviews`)
- **Блог** (`/blog`)
- **Калькулятор** (`/calculator`)

## 🔧 Как исправить проблему:

### Вариант 1: Автоматически (рекомендуется)

После следующего деплоя seed автоматически создаст все пункты меню. Если меню все еще пустое, используйте fallback меню (уже добавлено в код).

### Вариант 2: Вручную через seed

**Локально:**
```bash
npm run prisma:seed
```

**В продакшене (через Vercel CLI):**
```bash
vercel env pull .env.local
export DATABASE_URL="your_postgresql_url"
npm run prisma:seed
```

### Вариант 3: Через админ-панель

1. Войдите в админку: `/admin/login`
2. Перейдите в **Настройки** → **Меню**
3. Убедитесь, что все пункты меню созданы и активны
4. Если пунктов нет, создайте их вручную:
   - Главная: `/`
   - Услуги: `/services`
   - Контакты: `/contacts`
   - Кейсы: `/cases`
   - Отзывы: `/reviews`
   - Блог: `/blog`
   - Калькулятор: `/calculator`

## 🔍 Проверка:

1. **Откройте сайт** и проверьте меню в шапке
2. **Проверьте футер** - там тоже должно быть меню
3. **Проверьте консоль браузера** (F12) - не должно быть ошибок
4. **Проверьте логи сервера** - должны быть сообщения о создании меню

## 📝 Что происходит:

1. **При загрузке страницы:**
   - Система пытается загрузить меню из базы данных
   - Если меню пустое или произошла ошибка, используется дефолтное меню
   - Все разделы сайта будут доступны

2. **При деплое:**
   - Seed автоматически создает все пункты меню
   - Если seed не выполнился, используется fallback меню
   - Сайт работает даже если база данных не заполнена

## ✅ После исправления:

1. Все разделы сайта должны быть видны в меню
2. Все ссылки должны работать
3. Меню должно быть одинаковым на локальном и продакшене

## 🐛 Если проблема сохраняется:

1. **Проверьте базу данных:**
   ```bash
   npm run prisma:studio
   ```
   Откройте таблицу `MenuItem` и проверьте, что все пункты созданы

2. **Проверьте переменные окружения:**
   - `DATABASE_URL` должен быть настроен
   - В Vercel должны быть все нужные переменные

3. **Проверьте логи:**
   - В консоли браузера (F12)
   - В логах Vercel (Dashboard → Deployments → Logs)

4. **Принудительно выполните seed:**
   ```bash
   npm run prisma:seed
   ```
