# 🚀 Быстрая настройка базы данных

## ⚠️ ВАЖНО: Сначала настройте DATABASE_URL!

**Без DATABASE_URL сайт не будет работать!**

📖 **Полная инструкция:** См. файл `НАСТРОЙКА-БАЗЫ-ДАННЫХ-VERCEL.md`

## 🎯 Быстрые шаги:

### 1. Создать базу данных в Vercel

1. Vercel Dashboard → ваш проект → **Storage** → **Postgres**
2. Нажмите **"Create Database"**
3. Выберите план (можно бесплатный)
4. Скопируйте **Connection String**

### 2. Добавить DATABASE_URL

1. Vercel Dashboard → **Settings** → **Environment Variables**
2. Добавьте:
   - **Key:** `DATABASE_URL`
   - **Value:** вставьте Connection String
   - **Environment:** Production, Preview, Development
3. Сохраните

### 3. Применить миграции и заполнить данные

**Вариант A: Через API (после деплоя)**
```
https://ваш-домен.vercel.app/api/admin/emergency-fix
```

**Вариант B: Через CLI (локально)**
```powershell
vercel link
vercel env pull .env.local
npx prisma migrate deploy
npx tsx prisma/seed.ts
```

### 4. Сделать Redeploy

1. Vercel Dashboard → **Deployments** → **Redeploy**
2. Дождитесь завершения

## ⚠️ Если что-то не работает:

### Вариант 1: Применить миграции вручную через Vercel CLI

```bash
# Установите Vercel CLI (если еще не установлен)
npm i -g vercel

# Подключитесь к проекту
vercel link

# Получите переменные окружения
vercel env pull .env.local

# Примените миграции
npx prisma migrate deploy

# Заполните базу данных
npx tsx prisma/seed.ts
```

### Вариант 2: Через Vercel Dashboard

1. Откройте проект в Vercel Dashboard
2. Перейдите в Settings → Environment Variables
3. Убедитесь, что `DATABASE_URL` настроен правильно
4. Запустите новый деплой

### Вариант 3: Через SQL клиент

1. Подключитесь к вашей PostgreSQL базе данных
2. Выполните SQL из файла `prisma/migrations/init/migration.sql`
3. Затем выполните seed: `npx tsx prisma/seed.ts`

## 📝 Примечания:

- **Локальная разработка**: Используется SQLite (файл `prisma/dev.db`)
- **Продакшен**: Используется PostgreSQL (настроенный в Vercel)
- **Миграции применяются автоматически** при каждом деплое
