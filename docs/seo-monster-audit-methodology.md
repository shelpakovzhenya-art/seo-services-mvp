# SEO Monster Audit Methodology

This note captures the audit rules we extracted from `SEO Монстр NEXT.pdf` on 2026-03-25 via local OCR and translated into the internal SEO audit generator.

## Source pages used

- PDF pages `291-300`: commercial ranking factors, delivery/payment, guarantees, support, chat, discounts, URL length, title relevance.
- PDF pages `545-556`: internal linking, topical relevance, silo / semantic cocoon, breadcrumb and anchor guidance.
- PDF pages `600-606`: content audit, consolidation / archive decisions, repeat optimization, proof-driven content refresh.

## Rules now enforced in the audit

### 1. Commercial intent is not satisfied by generic SEO text

Priority pages should be checked for:

- price or price range
- delivery / payment or work process
- guarantees / support
- visible contact points
- trust layer: reviews, cases, certificates, proof

If this layer is missing on a large share of priority pages, the audit should raise a high-priority issue instead of praising the site for generic technical hygiene.

### 2. Content quality means proof, not volume

The audit should flag pages that are:

- thin
- missing FAQ / reviews / comparison blocks
- missing evidence and proof elements

Recommended actions should follow the book logic:

- find lost queries
- expand with new H2/H3
- add facts, lists, comparisons, cases, screenshots, media
- update snippet fields
- relink the refreshed page

### 3. Internal linking must support clusters, not just navigation chrome

The audit should treat internal linking as a ranking and UX layer:

- important pages should sit within a few clicks from the homepage
- links should be topically relevant
- anchors should describe the destination
- breadcrumbs and "related" blocks should support the hierarchy
- service/query noise should not compete with landing pages

### 4. Content cleanup is part of SEO, not just publishing

For future development, the audit should keep the book model in mind:

- leave strong pages as is
- rewrite outdated but valuable pages
- merge overlapping pages
- archive / remove deadweight content that brings no traffic, links or value

### 5. Snippets must stay honest

Title and description should not promise benefits that the page does not actually show. Misaligned snippets create pogo-sticking risk and damage both CTR quality and ranking resilience.

## Practical implication for generator output

The report should avoid:

- generic "strengths" that clients did not buy the audit for
- vague reassurance without proof
- abstract growth advice detached from detected page signals

The report should prefer:

- evidence-backed issues
- priority-page examples
- commercial-factor gaps
- repeat-optimization recommendations
- topical internal-linking fixes
