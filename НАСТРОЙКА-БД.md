# Настройка базы данных для продакшена

## ✅ Что уже сделано:

1. **Схема Prisma обновлена** - теперь использует PostgreSQL для продакшена
2. **Миграция создана** - файл `prisma/migrations/init/migration.sql` содержит все таблицы
3. **Скрипт setup.js обновлен** - автоматически применяет миграции в продакшене

## 📋 Что нужно сделать:

### 1. Убедитесь, что DATABASE_URL настроен в Vercel

В настройках проекта Vercel должна быть переменная окружения:
```
DATABASE_URL=postgresql://user:password@host:port/database
```

### 2. Применить миграции вручную (если нужно)

Если миграции не применились автоматически при деплое, выполните вручную:

**Вариант 1: Через Vercel CLI (рекомендуется)**
```bash
vercel env pull .env.local
npx prisma migrate deploy
```

**Вариант 2: Через Prisma Studio (онлайн)**
1. Откройте Prisma Studio: `npx prisma studio`
2. Подключитесь к вашей базе данных
3. Проверьте, что все таблицы созданы

**Вариант 3: Через SQL клиент**
1. Подключитесь к вашей PostgreSQL базе данных
2. Выполните SQL из файла `prisma/migrations/init/migration.sql`

### 3. Заполнить базу данных начальными данными

После применения миграций, выполните seed:

```bash
# Установите DATABASE_URL для продакшена
export DATABASE_URL="postgresql://user:password@host:port/database"

# Запустите seed
npx tsx prisma/seed.ts
```

Или через Vercel CLI:
```bash
vercel env pull .env.local
npx tsx prisma/seed.ts
```

## 🛠️ Локальная разработка

Для локальной разработки используется SQLite (файл `prisma/dev.db`).

Если хотите использовать PostgreSQL локально:
1. Установите PostgreSQL
2. Создайте базу данных
3. Обновите `.env`:
   ```
   DATABASE_URL="postgresql://user:password@localhost:5432/database"
   ```
4. Примените миграции: `npx prisma migrate deploy`

## 📝 Примечания

- **Миграции применяются автоматически** при деплое на Vercel через `scripts/setup.js`
- **Seed выполняется автоматически** в продакшене
- **Локально** используется SQLite для удобства разработки
- **В продакшене** используется PostgreSQL (настроенный в Vercel)

## ❓ Проблемы?

Если что-то не работает:

1. Проверьте `DATABASE_URL` в Vercel
2. Проверьте логи деплоя в Vercel
3. Попробуйте применить миграции вручную
4. Проверьте подключение к базе данных
