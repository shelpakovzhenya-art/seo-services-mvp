# 📋 Пошаговая инструкция для исправления

## 🎯 Проблема:
- Ссылка не изменилась (домен не работает)
- Разделы не добавились (меню пустое)
- Обновления залиты на GitHub, но не применились

## ✅ Решение (пошагово):

### Шаг 1: Проверьте деплой в Vercel

1. Откройте [Vercel Dashboard](https://vercel.com/dashboard)
2. Найдите ваш проект `seo-services-mvp`
3. Проверьте последний деплой:
   - Должен быть статус **"Ready"** (зеленый)
   - Если статус "Error" или "Building" - подождите или нажмите "Redeploy"

### Шаг 2: Настройте переменные окружения

**Это самое важное!**

1. В Vercel Dashboard → ваш проект → **Settings** → **Environment Variables**
2. Проверьте/добавьте эти переменные:

   ```
   DATABASE_URL=postgresql://user:password@host:port/database
   ```
   (Ваша строка подключения к PostgreSQL)

   ```
   SITE_URL=https://seo-update.ru
   ```
   (Или ваш текущий домен Vercel, например: `https://seo-services-mvp.vercel.app`)

   ```
   ADMIN_USER=admin
   ADMIN_PASS=admin123
   ```

3. **ВАЖНО:** После добавления/изменения переменных - нажмите **"Redeploy"** на последнем деплое!

### Шаг 3: Заполните базу данных

**Способ 1: Через Vercel CLI (рекомендуется)**

```bash
# 1. Установите Vercel CLI (если еще не установлен)
npm i -g vercel

# 2. Войдите в Vercel
vercel login

# 3. Подключитесь к проекту (выберите существующий проект)
vercel link

# 4. Получите переменные окружения
vercel env pull .env.local

# 5. Проверьте, что DATABASE_URL есть в .env.local
# Откройте .env.local и проверьте DATABASE_URL

# 6. Запустите seed (заполнит базу данных)
npx tsx prisma/seed.ts
```

**Способ 2: Через админ-панель (если можете войти)**

1. Откройте ваш сайт: `https://your-site.vercel.app/admin/login`
2. Войдите: `admin` / `admin123`
3. Перейдите в **Настройки** → **Меню**
4. Создайте пункты меню вручную:
   - Нажмите "Добавить пункт меню"
   - Главная: `/` (порядок: 1)
   - Услуги: `/services` (порядок: 2)
   - Контакты: `/contacts` (порядок: 3)
   - Кейсы: `/cases` (порядок: 4)
   - Отзывы: `/reviews` (порядок: 5)
   - Блог: `/blog` (порядок: 6)
   - Калькулятор: `/calculator` (порядок: 7)

### Шаг 4: Настройте домен (если нужно seo-update.ru)

1. В Vercel Dashboard → Settings → **Domains**
2. Нажмите **"Add Domain"**
3. Введите: `seo-update.ru`
4. Vercel покажет DNS записи, которые нужно добавить у регистратора домена
5. Добавьте DNS записи у регистратора домена
6. Дождитесь распространения DNS (1-24 часа)
7. После этого обновите `SITE_URL` в переменных окружениях на `https://seo-update.ru`
8. Сделайте **Redeploy**

### Шаг 5: Принудительный редеплой

Если изменения не применились:

1. В Vercel Dashboard → **Deployments**
2. Найдите последний деплой
3. Нажмите **"..."** → **"Redeploy"**
4. Или сделайте пустой коммит:
   ```bash
   git commit --allow-empty -m "Force redeploy"
   git push
   ```

## 🔍 Проверка:

### Проверьте меню:
1. Откройте сайт
2. В шапке должно быть меню: Главная, Услуги, Контакты, Кейсы, Отзывы, Блог, Калькулятор
3. В футере тоже должно быть меню

### Проверьте домен:
1. Если настроили `seo-update.ru` - проверьте, что он открывается
2. Проверьте SSL (должен быть зеленый замочек)

### Проверьте админку:
1. Войдите: `/admin/login`
2. Проверьте, что можете войти
3. Проверьте настройки → меню

## ⚠️ Важные моменты:

1. **После изменения переменных окружения - ОБЯЗАТЕЛЬНО сделайте Redeploy!**
2. **DATABASE_URL должен быть правильным** - проверьте строку подключения
3. **Seed должен выполниться** - либо автоматически при деплое, либо вручную
4. **DNS для домена** может занять до 24 часов

## 🆘 Если не работает:

1. **Проверьте логи:**
   - Vercel Dashboard → Deployments → последний деплой → Logs
   - Ищите ошибки

2. **Проверьте подключение к базе:**
   - Убедитесь, что DATABASE_URL правильный
   - Проверьте, что база данных доступна из интернета

3. **Проверьте консоль браузера:**
   - F12 → Console
   - Ищите ошибки
